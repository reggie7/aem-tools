This component illustrates current basic usage of i18n within MIM systems. It contains 2 separate sections showing basic usage scenarios.

Scenario #1 - a fake shared label. A component that is used only once within a language branch.

${'maark.dashboard.text' @ i18n}

Scenario #2 - truly shared labels. One can imagine that it makes a lot of sense to extract common button labels and NOT to set them each time again and again.

${'maark.button.yes' @ i18n} ${'maark.button.no' @ i18n}



Note how there is no dialog - everything is configured in AEM Translator within dictionaries. Let's see the workflow now maybe...
