Below you will find examples of message boxes usage

This is
a sample table
You achieve it with author-table class
Margin comes from author-margin class

Edit This is a box meant to tell the author there's something to be edited in the current component. You can achieve it with author-msg-box author-edit classes. These are independent to some extent and can be replaced.

Error This is a line meant to tell the author there's seriously something wrong or an operation failed. We achieved this by replacing the previous classes by author-msg-line author-error respectively. The first one changed the size, the second - colors and icon.

Success This is a narrow line meant to tell the authors their operation succeeded. We used author-msg-narrow author-success respectively. The first one minimizes the size as you can see hopefully.

This is a line meant to warn the author.

We used author-msg-paragraphs author-warning respectively.

The first one allows us to use paragraphs with more messages.

The underlying background comes from author-preview-box class. You can use it to indicate the content is a preview of something.

An example would be a popup content that cannot be easily shown within author's environment.

Previewed Content's Title Preview can display an icon when used with a span element just as in here.

Notice This is a box meant to show a simple info. We used author-msg-box author-notice.

Component This is a box meant to show a component that has no visible content when published. We used author-msg-box author-component.

Reference This is a box meant to show a reference indication. We used author-msg-box author-reference.

Configuration This is a box meant to show a configuration component without directly visible content when published. We used author-msg-box author-configuration.
